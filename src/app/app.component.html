Redes II

Funciones del router Un router es un dispositivo de red que dirige el tráfico de datos entre diferentes redes. Sus principales funciones incluyen:
  • Una red remota es aquella que no está directamente conectada a un router.
  • Para alcanzar una red remota, el router debe tener una ruta que especifique cómo llegar a ella.
  • En el routing estático, el administrador configura rutas específicas para alcanzar redes remotas, utilizando direcciones IP y métricas predefinidas.
 Configuración básica del router Los pasos básicos para configurar un router incluyen:
  • Acceso a la interfaz de administración: Normalmente se hace a través de un navegador web ingresando la dirección IP del router (como 192.168.1.1).
  • Cambio de credenciales predeterminadas: Es recomendable modificar el usuario y la contraseña por seguridad.
  • Configuración de la red Wi-Fi: Definir un nombre de red (SSID) y una contraseña segura (preferiblemente WPA2 o WPA3).
  • Asignación de direcciones IP: Configurar el DHCP si se desea que el router asigne direcciones IP automáticamente.
  • Ajustes de seguridad: Activar el firewall, deshabilitar WPS si no es necesario y filtrar direcciones MAC si se requiere control de acceso.
 Verificación de la conectividad de redes conectadas directamente Para comprobar si la red está funcionando correctamente, se pueden realizar los siguientes pasos:
  • Revisión de indicadores LED del router: Verificar que las luces de conexión Ethernet, Wi-Fi e Internet estén activas.
  • Comprobación de conectividad con comandos: Usar comandos como ping (ej. ping 8.8.8.8) o tracert en Windows/Linux para verificar la conexión con otros dispositivos o con Internet.
  • Acceso a la configuración del router: Revisar la lista de dispositivos conectados y verificar que tengan direcciones IP asignadas.
  • Pruebas de velocidad: Utilizar herramientas como Speedtest para medir la velocidad de conexión.
  • Prueba de conexión en diferentes dispositivos: Si un solo dispositivo no tiene acceso, podría haber un problema de configuración en ese equipo específico.
 Switching de paquetes entre redes El switching de paquetes se refiere al proceso de transferencia de paquetes desde una red de origen a una red de destino. Existen varios métodos de switching:
  • Switching basado en procesos: Cada paquete es examinado y ruteado individualmente.
  • Switching rápido (Fast Switching): Se almacena en caché la información de rutas para mejorar la eficiencia.
  • Switching CEF (Cisco Express Forwarding): Utiliza una tabla preconstruida para tomar decisiones de forwarding de manera más eficiente.
 Determinación de ruta Los routers eligen la mejor ruta basada en diferentes métricas y algoritmos, incluyendo:
  • Métricas: Ancho de banda, retraso, carga, confiabilidad y número de saltos.
  • Algoritmos de routing:
    • Vector distancia (RIP, IGRP): Basados en la cantidad de saltos.
    • Estado de enlace (OSPF, IS-IS): Basados en el estado de los enlaces y cálculo de la mejor ruta con Dijkstra.
    • Híbridos (EIGRP): Combinan características de ambos tipos.
 Análisis de la tabla de routing La tabla de routing es una base de datos que almacena rutas conocidas y sus métricas. Se compone de:
  • Red de destino: Identifica la red a la que pertenece una dirección IP.
  • Máscara de subred: Determina la porción de red y host en una dirección IP.
  • Puerta de enlace: Dirección del siguiente salto hacia el destino.
  • Interfaz de salida: La interfaz por la que se enviará el paquete.
  • Métrica: Indica la preferencia o costo de la ruta.
 Rutas conectadas directamente Son aquellas rutas que el router conoce porque tienen una interfaz configurada dentro de esa red. Se identifican en la tabla de routing con una 'C' (Connected). Rutas descubiertas estáticamente Las rutas estáticas son configuradas manualmente por el administrador de red y permanecen fijas en la tabla de routing. Se identifican con una 'S' (Static) y pueden ser útiles en redes pequeñas o para definir rutas de respaldo. Ejemplo de configuración de una ruta estática en Cisco:
Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.1 Protocolos de routing dinámico Los protocolos de routing dinámico permiten a los routers intercambiar información de rutas automáticamente. Se clasifican en:
  • RIP (Routing Information Protocol): Basado en vector distancia, usa el número de saltos como métrica.
  • OSPF (Open Shortest Path First): Basado en estado de enlace, calcula la mejor ruta mediante Dijkstra.
  • EIGRP (Enhanced Interior Gateway Routing Protocol): Protocolo híbrido, más eficiente que RIP y con menor consumo de recursos que OSPF.
  • BGP (Border Gateway Protocol): Utilizado en redes de gran escala, como Internet.
Routing Estático El routing estático es un método de enrutamiento en el que un administrador configura manualmente las rutas en un router. A diferencia del enrutamiento dinámico, las rutas estáticas no cambian automáticamente en respuesta a cambios en la red. Alcance de las Redes Remotas
  • Una red remota es aquella que no está directamente conectada a un router.
  • Para alcanzar una red remota, el router debe tener una ruta que especifique cómo llegar a ella.
  • En el routing estático, el administrador configura rutas específicas para alcanzar redes remotas, utilizando direcciones IP y métricas predefinidas.
 Utilizar el Routing Estático Para implementar el routing estático se deben seguir los siguientes pasos:
  • Determinar la dirección de la red de destino y su máscara de subred.
  • Definir la dirección IP de la interfaz de salida o el siguiente salto.
  • Configurar la ruta estática en el router.
  • Verificar la conectividad y la correcta configuración de la ruta.
Comando básico en Cisco IOS:
Router(config)# ip route [red_destino] [máscara] [siguiente_salto o interfaz] Ejemplo:
Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.2 Usar Rutas Estáticas Las rutas estáticas son utilizadas en diversos escenarios:
  • Cuando se requiere una configuración estable sin cambios dinámicos.
  • Para redes pequeñas donde el mantenimiento de rutas es fácil.
  • Como respaldo de protocolos de enrutamiento dinámicos.
  • Para mejorar la seguridad, ya que evita la publicidad de rutas automáticas.
